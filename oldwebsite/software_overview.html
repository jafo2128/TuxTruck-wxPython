Realtime

Some amount of (soft) realtime will be required. Even if the unit is downloading weather data, navigating in realtime, and playing an MP3, if a phone call comes in, it needs to handle that call in a timely fashion (i.e., at a maximum, alert the user to that call within half a second). Of course, it's also possible to handle all of this in Python through good use of loops (possibly threads) and some sort of IPC facility.

One of the major considerations here is license terms. If the project goes well, I'd consider selling complete systems. If I do that, I need to make some sort of profit, to cover the time needed to build and test the systems, as well as cover the capital of buying parts. Therefore, a license that allows commercial use without any cost (I can't buy a license if I end up only selling 2 systems) is required.

Linux GPS Navigation Software

  • RoadNav / LibRoadNav - seems like the front-runner. Libroadnav is coded in C++/wxWindows. Missing a lot of features that I want, but seems like the leader. Provides Libroadnav (C++) for embedding in applications.
  • Navit - GTK+, designed for touchscreens. Has a *very* good UI. Uses speechd. Unfortunately written in C.
